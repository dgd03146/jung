# 매직 넘버에 이름 붙이기

## 가독성
매직 넘버(Magic Number)란 **정확한 뜻을 밝히지 않고 소스 코드 안에 직접 숫자 값을 넣는 것**을 말해요.

예를 들어,
- 찾을 수 없음(Not Found)을 나타내는 HTTP 상태 코드로 `404` 값을 바로 사용하는 경우
- 하루를 나타내는 `86400`초를 그대로 사용하는 경우

등이 있어요.

---

## 📝 코드 예시

다음 코드는 **좋아요 버튼을 눌렀을 때 좋아요 개수를 새로 내려받는 함수**예요.

```ts
async function onLikeClick() {
  await postLike(url);
  await delay(300);
  await refetchPostLike();
}


⸻

👃 코드 냄새 맡아보기

가독성

이 코드에서는 delay 함수에 전달된 300이라는 값이 어떤 맥락으로 쓰였는지 알 수 없어요.
원래 코드를 작성한 개발자가 아니라면, 왜 300ms 동안 기다리는지 이해하기 어렵죠.

다음과 같은 의문이 생길 수 있어요.
	•	애니메이션이 완료될 때까지 기다리는 걸까?
	•	좋아요 반영에 시간이 걸려서 기다리는 걸까?
	•	테스트 코드였는데, 깜빡하고 안 지운 걸까?

하나의 코드를 여러 명의 개발자가 함께 수정하다 보면,
의도를 정확히 알 수 없어서 코드가 원하지 않는 방향으로 수정될 수도 있어요.

INFO
이 함수는 응집도 관점으로도 볼 수 있어요.

⸻

✏️ 개선해보기

숫자 300의 맥락을 정확하게 드러내기 위해
상수 ANIMATION_DELAY_MS로 선언할 수 있어요.

const ANIMATION_DELAY_MS = 300;

async function onLikeClick() {
  await postLike(url);
  await delay(ANIMATION_DELAY_MS);
  await refetchPostLike();
}

ANIMATION_DELAY_MS를 함수 밖에 선언해서 사용하는 것도 아주 좋아요.
이렇게 하면 함수가 호출될 때마다 새로 선언되는 것이 아니라,
한 번만 선언되어 재사용되기 때문에 더 바람직해요.

여기서 이런 패턴을 다시 보니 반갑네요 ☺️

💡 추가로 알아두면 좋은 팁: Numeric Separators

안녕하세요. 글 잘 읽었습니다.
한 가지 덧붙이고 싶은 내용이 있어요.

숫자 리터럴에 언더바(_)를 사용하는 숫자 구분 기호(Numeric Separators) 스펙이
ECMAScript 2021에 추가되었어요.
	•	Node v14.13.0 이상
	•	Chrome 91 이상

즉, 2021년 이후의 브라우저라면 대부분 사용 가능한 스펙이에요.

가독성 관점에서 함께 사용하면 좋아요.

const ONE_MILLION = 1_000_000; // 1000000과 동일
const BINARY_TEN = 0b0000_1010; // 2진수에서도 사용 가능
const HEX_TEN = 0x00_00_0A; // 16진수에서도 사용 가능

숫자의 의미를 이름으로 드러내고,
숫자 자체도 읽기 쉽게 만드는 습관은
장기적으로 유지보수에 큰 도움이 돼요.
